Gunicorn

A python HTTP WSGI Server for UNIX. It allows you to run any python application concurrently by running multiple python processes(multi-threaded).
It the most prefered server working with frameworks like Django and Flask etc. in production.

Installation

We assume that our project is been setup in a vitual environment so lets install gunicorn in our project virtual environment.

Testing Gunicorn

After gunicorn has been successfully installed in our project virtual environment we will have to test it and confirm if it can server our project.

NB: The test was referenced with a Django project.

We passed a Gunicorn a module by specifying the relative directory path to the Django's wsgi file.

You can test it by visiting the URL http://your-server-ip:8000.

If the test is successfully, gunicorn will be running on the same interface[port] that the django development server was runnin on.

Once the test is successful, we can stop Gunicorn by pressing CTRL-C in terminal window.

Deactivate Virtual Environment

Once our testing is complete we have to deactivate our virtual environment to configure and setup our UNIX machine to run Gunicorn

Creating systemd Socket file for Gunicorn

Since the testing have proven we can run our project with Gunicorn, we have to implement a more robust way of starting and stoping the application server outside of the V.E

A Gunicorn socket will be created at boot and will listen for connections. When a connection is establish, systemd will automatically start Gunicorn process to handle the process.

Lets start by creating and opening systemd socket file for Gunicorn with sudo privileges

We are going to create 3 section in the file:

/etc/systemd/system/gunicorn.socket

[Unit]
Description=gunicorn socket

[Socket] ListenStream=/run/gunicorn.sock

[Install]
WantedBy=sockets.target

[Unit] section describes the socket

[Socket] section defines the socket location

[Install] section makes sure the socket is created at the right time

Now Save and close the file with the nano command.
